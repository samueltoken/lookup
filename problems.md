# Problems


## [2026-04-15] v1.2.5 착수 전 이슈
- HWP 보안 접근 허용 팝업이 반복되어 사용 흐름이 자주 끊김.
- 일부 HWP 변환에서 책자형(예: 8|1) 페이지 결합 결과가 발생.
- XLSX가 엔진 실패 시 읽기 어려운 fallback만 표시됨.
- 보기 모드(단면/양면/현재 페이지) 요구 대비 UI 상태 저장/전환 연결 미완.

## [2026-04-15] lookup v1.2.6 착수 전 재발/개선 요청
- HWP 변환 결과에서 원본 대비 비정상 가로줄(줄쳐진 것처럼 보이는 선) 현상이 보고됨.
- 키워드 검색 하이라이트가 단어 경계를 벗어나 칠해져 정확도 불만이 재발함.
- 주석 도구와 설정 항목의 위치가 사용 흐름과 맞지 않아 한 번 클릭 동선이 깨짐.
- XLSX 다중 시트 파일에서 시트 단위 이동이 불가능해 긴 문서 탐색이 어려움.

## [2026-04-15] lookup v1.2.7 착수 전 핵심 원인
- HWP 인쇄형 변환 함수에서 Print 옵션을 설정한 뒤 다시 기본값을 불러오는 흐름이 있어 옵션이 되돌아갈 가능성이 확인됨.
- 잘못 생성된 HWP 변환 PDF가 캐시에 남아 재열기 시 동일 문제가 반복될 수 있음.
- 단일 변환 경로만 신뢰해 품질 비교 없이 결과를 확정하는 구조라 줄 아티팩트가 있는 출력이 채택될 수 있음.

## [2026-04-15] lookup v1.2.8 착수 전 재발 이슈
- 특정 HWP(붙임 1 공고문.hwp)에서 줄 아티팩트가 재발했고, 기존 후보 선택 기준이 강하지 않아 문제 출력이 선택될 여지가 있었음.
- HWP 열기 경로에서 COM 변환/품질 점검이 동기 방식으로 묶여 있어 체감 로딩 지연이 발생함.
- 변환 결과가 좋아졌는지 빠르게 재사용할 선택 메타 캐시가 없어 재열기 때도 분석 비용이 반복됨.

## [2026-04-15] lookup v1.3.0 착수 전 핵심 이슈
- Hancom/MS Office가 없는 PC에서 `doc/xls/pptx` 열기가 엔진 의존으로 자주 실패해 완전 오프라인 열람 성공률이 낮았음.
- HWP 변환 결과가 파일 존재만으로 캐시 채택되어, 깨진 PDF(`Invalid PDF structure`)가 선택되면 흰 화면이 재발했음.
- HWP COM 호출 스크립트의 경로 배열 문법 오류로 native 변환 후보 생성이 실패할 가능성이 있었음.
- `ppt/pptx`가 파일 열기 UI/설치 연결 확장자에서 빠져 있어 사용 흐름이 끊겼음.
- 변환 단계 진행 상태 이벤트가 renderer까지 연결되지 않아 사용자에게 현재 단계가 충분히 보이지 않았음.

## [2026-04-15] lookup v1.3.1 착수 전 재발 이슈
- HWP 파일 열기 시 한컴 보안 접근 허용 팝업이 자주 떠서 사용자 흐름이 끊김.
- 일반 HWP 열기 중에도 내부 Print 경로가 개입되어 "인쇄 중" 창이 뜨는 문제가 반복됨.
- 한컴 COM 예외/객체 참조 오류창이 사용자 화면에 그대로 노출됨.
- 업데이트 완료 팝업에서 버전과 변경 항목 계층이 섞여 가독성이 낮음.
