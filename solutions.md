# Solutions


## [2026-04-15] v1.2.5 개선 적용
- HWP 보안 모듈 자동 점검/등록 시도를 추가하고 실패 시 fallback으로 자동 전환.
- HWP 결과가 책자형으로 의심되면 hwpjs 재변환을 1회 자동 시도.
- 보기 모드 3종(연속 단면/연속 양면/현재 페이지만) 버튼을 리본/전체화면에 연결하고 상태 저장.
- XLSX HTML 표 fallback 경로를 적용해 시트 기반 열람성 개선.
- 릴리즈/메시지 파일을 UTF-8로 정리해 한글 깨짐 재발 완화.

## [2026-04-15] lookup v1.2.6 개선 적용
- HWP 변환 기본 경로를 한컴 COM 인쇄형 PDF로 고정하고, 인쇄 프로파일에서 검토/메모/표시선 계열 옵션을 비활성화해 줄 아티팩트 발생 가능성을 낮췄습니다.
- 검색 문자열을 원문 인덱스와 매핑하는 방식으로 바꾸고, 하이라이트를 단어 내부 밴드로 그려 단어 밖으로 번지는 현상을 줄였습니다.
- 주석 기능(형광펜/펜/텍스트 메모/되돌리기/다시하기)을 도구 탭으로 통합하고, 업데이트/문의/언어/버전은 설정 메뉴에서 바로 실행되게 정리했습니다.
- XLSX 변환 결과에 시트 메타데이터(sheetMap)를 포함하고, 리본 시트 선택 UI에서 시트별 시작 페이지로 즉시 이동할 수 있게 보강했습니다.
- 추가 안정화로 제거된 빠른 설정 버튼 참조를 정리해 런타임 오류 가능성을 낮췄습니다.

## [2026-04-15] lookup v1.2.7 개선 적용
- HWP 인쇄형 변환에서 Print 옵션 설정 후 기본값을 다시 불러오던 흐름을 제거해 옵션 덮어쓰기 재발을 차단했습니다.
- HWP 변환 결과를 인쇄형/저장형 두 후보로 생성하고, 가로줄 겹침/텍스트 가독성/책자형 패턴을 점수화해 더 좋은 결과를 자동 선택하도록 변경했습니다.
- HWP 캐시 스키마를 갱신(`v4`)하고 품질 선택 버전을 캐시 키에 포함해 이전 잘못된 변환 결과 재사용 가능성을 낮췄습니다.
- `document:open`과 `document:convert` 응답에 `convertDiagnostics`를 추가해 선택된 변환 모드와 후보 점수를 확인할 수 있게 했습니다.

## [2026-04-15] lookup v1.2.8 개선 적용
- HWP 선택 엔진을 v2(균형형)로 바꿔 `빠른 print 점검 → 이상 징후 시 saveas/full 비교 → 필요 시 hwpjs` 순서로 동작하게 했습니다.
- HWP 선택 메타 캐시(선택 모드/점수/파일 서명)를 별도 파일로 저장해, 같은 파일 재열기 시 재분석을 건너뛰고 즉시 열 수 있게 했습니다.
- 품질 분석에 `rasterLineScore` 계열 지표(가로줄 관통 패턴 추정)를 추가하고 최종 점수를 `vector + raster` 통합 점수로 계산하도록 강화했습니다.
- HWP/Office COM 변환 함수를 비동기 실행으로 전환해 UI 멈춤을 줄이고, 단계별 진단 로그를 남겨 병목 추적이 가능하게 했습니다.
- `convertDiagnostics`에 `selectorVersion`, `analysisMode`, 상세 후보 점수(`vectorLineScore`, `rasterLineScore`, `finalScore`)를 포함해 원인 확인이 쉬워졌습니다.
